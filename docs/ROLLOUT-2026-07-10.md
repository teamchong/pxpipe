# PXpipe rollout decision record — 2026-07-10

## Decision

Accept the partial council synthesis plus the short follow-up debate as the
working rollout verdict for V1. This is not recorded as a formal council
`FinalVerdict`; it is the implementation decision for this repo until a future
formal council run replaces it.

## Order

1. Freeze the live baseline with tag `v1-baseline-2026-07-10`.
2. Add a live build/runtime fingerprint endpoint (`GET /version`) and `pxpipe doctor`.
3. Extend read-only telemetry for `tier0_dropped_total`, 400/refusal rows,
   cache-prefix stability, baseline-probe status, omitted chars, and event schema.
4. Correct render-sizing docs to the current 1568x728 page geometry.
5. Wire C9/C10 request-weight classification as shadow-only telemetry,
   default OFF via `PXPIPE_ROUTING_SHADOW`.
6. Strengthen factsheet abstention wording and keep `tier0DroppedTotal` as the
   cheap fidelity signal before any retrieval/stash work.
7. Keep B6 retrieval-on-demand gated behind telemetry proof, a separate flag,
   canary, typed contracts, TTL/GC, and a dedicated security review.

## Non-decisions

- C9/C10 does not change routing in this rollout.
- B6 stash/retrieval is not implemented in this rollout.
- The old recover channel remains test-only/dead in production unless B6 is
  approved as a separate project.

## Observation helper

After PR #98 is merged and PXpipe is restarted, collect the 48h evidence with:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/observe-rollout.ps1
```

This captures T0, T+24h, and T+48h snapshots into
`~/.pxpipe/rollout-observations/pr-98/` and writes `conclusions.md`.

If the operator wants C9/C10 data, restart PXpipe with
`PXPIPE_ROUTING_SHADOW=1` before T0. This only logs `routing_shadow_*`; it does
not change live routing.

For a watcher that waits until the PR is merged before taking T0:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/observe-rollout.ps1 -WatchForMerge
```
